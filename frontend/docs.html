Everything you need to integrate, deploy, and understand the medical-grade skin cancer diagnostic API.

Quickstart

Get started with Skinderm AI in less than 5 minutes. Follow these three steps to integrate our diagnostic engine into your workflow.

1
Generate API Key: Go to the Dashboard and create a new project key.
2
Prepare Image: Ensure the lesion is centered and well-lit (see Best Practices).
3
Call Endpoint: Send a POST request to /v1/analyze with your image file.

Imaging Best Practices

To ensure maximum diagnostic accuracy (up to 98%), please adhere to the following imaging standards:

  • Lighting: Use natural daylight or diffuse LED lighting. Avoid direct camera flash which causes glare on skin surface.
  • Focus: Use macro-mode if available. The lesion must be in sharp focus. Blurry images will be rejected by the pre-processing layer.
  • Background: Place the lesion against a neutral skin background. Remove hair if possible using surgical tape or scissors.
  • Scale: Include a reference scale (ruler or coin) near the lesion for size estimation.

Authentication

All API requests must be authenticated using a Bearer token. You can generate an API key from your Skinderm dashboard.

cURL 
curl -X POST https://api.skinderm.ai/v1/analyze \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: multipart/form-data" \
  -F "file=@/path/to/lesion.jpg"

API Endpoints

Our REST API provides a simple interface for diagnostic analysis.

POST
/v1/analyze

Analyzes a dermatological image and returns risk scores, ABCDE metrics, and a diagnostic classification.

Response Example

JSON 
{
  "status": "success",
  "data": {
    "classification": "Melanoma",
    "risk_score": 89.5,
    "abcde": {
      "A_asymmetry": { "score": 85, "status": "High Risk" },
      "B_border": { "score": 72, "status": "Irregular" },
      "C_color": { "score": 91, "status": "Variegated" }
    }
  }
}

Model Architecture

Skinderm uses a custom-trained ConvNeXtV2 architecture, optimized for high-resolution 448x448px inputs. This allows the network to identify microscopic cellular patterns such as atypical pigment networks or blue-white veils.

Explainable AI (XAI)

Trust is critical in medical AI. Skinderm utilizes Grad-CAM (Gradient-weighted Class Activation Mapping) to highlight the exact visual features that led to a specific diagnosis.

Transparency First

Every analysis result returns a heatmap overlay. If the AI flags a lesion as "High Risk", you can visually confirm if it's focusing on the lesion or accidental artifacts like hair or shadows.

Data Privacy (HIPAA/GDPR)

Your data security is our top priority. We implement enterprise-grade protection for all PHI (Protected Health Information).

  • End-to-End Encryption: All images are encrypted using AES-256 both at rest and in transit (TLS 1.3).
  • Automatic Anonymization: Our pre-processing pipeline automatically crops images to the lesion area, removing any potentially identifying features (faces, tattoos, background environment).
  • No-Log Policy: We do not store clinical images by default unless explicitly requested for longitudinal tracking by the healthcare provider.
